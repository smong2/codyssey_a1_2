# 🐳 도커(Docker) 기반 개발 및 실행 가이드

이 프로젝트는 개발 환경의 일관성을 유지하고 로컬(호스트) PC의 환경 오염을 방지하기 위해 **Docker**와 **Docker Compose**를 기반으로 구성되었습니다.

## 1. 도커 환경의 장점 및 특징

제공된 `Dockerfile`과 `docker-compose.yml`은 원활한 로컬 개발(Local Development)에 최적화되어 있습니다.

- **실시간 코드 동기화 (Volume Binding)**: 로컬 호스트의 `../src` 폴더가 컨테이너의 `/app/src`와 양방향으로 연결(Mount)되어 있습니다. 코드를 수정하면 이미지를 다시 빌드할 필요 없이 컨테이너 환경에 즉시 반영됩니다.
- **환경 변수 자동 주입**: `env_file` 설정을 통해 `../src/.env` 파일에 작성된 API 키들이 컨테이너 구동 시 자동으로 주입됩니다.
- **CLI 입출력 지원**: `tty: true`와 `stdin_open: true` 옵션이 켜져 있어, 도커 컨테이너 내부에서도 대화형 터미널이나 에러 메시지 출력을 완벽하게 확인할 수 있습니다.
- **경로(Path) 이슈 해결**: `PYTHONPATH=/app/src` 환경 변수가 설정되어 있어, 복잡한 경로 설정 없이 파이썬 모듈들을 자유롭게 Import 할 수 있습니다.

## 2. 사전 준비 (Prerequisites)

- 로컬 PC에 [Docker](https://www.docker.com/products/docker-desktop/) 및 Docker Compose가 설치되어 있어야 합니다.
- 프로젝트 루트 디렉토리 구조는 다음과 같아야 합니다.
  ```
  프로젝트_루트/
  ├── docker/
  │   ├── Dockerfile
  │   ├── docker-compose.yml
  │   └── requirements.txt
  └── src/
      ├── .env              # API 키 설정 파일 (필수)
      ├── main.py
      ├── api_service.py
      └── ...

  ```

## 3. 실행 방법

도커 환경에서는 컨테이너를 먼저 띄운(Up) 뒤, 해당 컨테이너 내부로 들어가서(Exec) CLI 명령어를 실행하는 방식으로 동작합니다.

### Step 1. 컨테이너 빌드 및 백그라운드 실행

`docker` 폴더가 있는 위치(또는 프로젝트 루트)에서 아래 명령어를 실행하여 컨테이너를 띄웁니다. *(※ 파일 경로에 따라* `-f` *옵션으로* `docker-compose.yml` *경로를 지정해야 할 수 있습니다.)*

```
# 프로젝트 루트 디렉토리 기준
docker-compose -f docker/docker-compose.yml up -d --build
```

### Step 2. CLI 프로그램 실행

`prompt-manager-app` 이라는 이름으로 실행된 컨테이너 내부에 파이썬 실행 명령어를 전달합니다.

```
docker exec -it prompt-manager-app python /app/src/main.py --date 2026-07-09
```

- `-it` 옵션: 로컬 터미널과 컨테이너 터미널을 연결하여 출력을 예쁘게(한글 깨짐 없이) 확인하기 위한 필수 옵션입니다.
- 정상적으로 실행되면 `src/results/` (로컬 디렉토리)에 리포트 결과물이 생성됩니다. 볼륨 마운트가 되어 있으므로 로컬 VS Code나 탐색기에서 바로 확인할 수 있습니다.

## 4. 종료 및 정리 방법

작업이 끝난 후 컨테이너를 종료하고 자원을 반환하려면 아래 명령어를 사용합니다.

```
# 컨테이너 종료 및 삭제
docker-compose -f docker/docker-compose.yml down
```
